## 💡요약

- Docker Compose와 Dockerfile에 관하여 알아봅니다.

## ✅ 요구사항

- 개인 레포지터리가 아닌 미리 생성된 레포지터리를 이용해주세요.
- 레포지터리에 바로 Commit 후 Push 하는게 아니라 자신의 이름으로 된 디렉터리를 하나 생성 후 그 안에 과제 번호(1-1,2-1,3-1…)를 이름으로 가지는 디렉터리 들을 생성하여 정리하도록 해주세요
- 각자 코드리딩을 진행하고 구현된 **5가지 엔드포인트(`GET /health/check`, `GET/api/v1/articles/all`, `POST /api/v1/articles`, `GET /api/v1/token/all`, `POST /api/v1/token`)에 대하여 Postman과 같은 프로그램으로 요청한 스크린샷을 PR에 첨부**하여 주세요
- **Dockerfile**를 작성하고 Github Action에서 통과하도록 하여주세요.
- Dockerfile 내부에서 Gradle 빌드를 진행하여 JAR 파일까지 생성하여야 합니다.
- master 브랜치가 아니라 **task/{자신의 이름}** 형태로 브랜치를 생성하여 master 브랜치를 향하여 PR을 걸어 주세요.
